All scripts that were used during development iterations:
- Legacy seeders (superseded by `comprehensive_seeder.py`)
- One-time fix scripts (already applied)
- Migration scripts (already applied)
- Test/debug scripts
- Data transformation scripts

## Do Reviewers Need This?

**No.** Reviewers only need the 3 scripts in the parent folder:
1. `comprehensive_seeder.py`
2. `generate_recipe_embeddings.py`
3. `seed_goals_direct.py`

## Why Keep Them?

Kept for reference only:
- Development history
- Reference if specific fixes are needed
- Transparency about development process

## Can I Delete This Folder?

Yes, if you want to reduce repository size further. These scripts are not needed for:
- Review
- Production
- Setup
- Basic functionality

But keeping them doesn't hurt and preserves development history.
