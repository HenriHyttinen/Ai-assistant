# Archived Documentation

This folder contains development documentation that was created during the development process but is no longer needed for reviewers or production.

## What's Here?

- **Root level docs**: Development summaries, optimization notes, compliance reports
- **Backend docs**: Completion reports, test results, deployment checklists
- **Development reports**: Issue reports, fix summaries, rebuild plans

## Do Reviewers Need This?

**No.** These are development notes and testing documentation. Reviewers only need:
- `README.md` (root and backend)
- `SETUP_FOR_REVIEWERS.md`
- `backend/docs/*.md` (technical documentation)
- `backend/tests/README.md`

## Why Archive Instead of Delete?

Kept for reference:
- Development history
- Transparency about development process
- Reference if specific issues come up

## Structure

```
_archive/
├── root/              # Root level development docs
├── backend_root/     # Backend root level development docs
└── backend_docs/     # Backend docs folder development reports
```
